## Required Site OS Rules

- Preserve factual accuracy.
- Do not invent reviews, ratings, license numbers, warranties, years in business, or business claims.
- Do not use Review or AggregateRating schema unless reviews are real, visible, and verified.
- FAQPage schema must match visible FAQ text exactly.
- HowTo schema is only allowed when visible step-by-step content exists.
- Customer-facing website copy must not use em dashes.
- Customer-facing website copy must not use double hyphens.
- Design changes must not alter copy unless explicitly requested.
- Do not change brand colors unless explicitly requested.
- All service business layouts should use a two-column hero where appropriate: content on the left, contact or quote form on the right.
- Service cards should include image placeholders as a standard.
- Every client website repo should include relevant Site OS prompts and documents needed to build optimized websites and optimized website content.
